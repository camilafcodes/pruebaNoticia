HT-04: Módulo de Consumo y Parseo de Feeds RSS
Título
Desarrollar el módulo de consumo y parseo de feeds RSS.
Descripción
Como sistema, quiero consumir y transformar los feeds RSS de una lista predefinida de fuentes a un formato JSON estandarizado, para poder procesar la información de cada noticia y asignarle su categoría correcta.
Responsable
Desarrollador Backend
Tareas Sugeridas
    1. Crear un módulo de configuración (ej. en un archivo config.js o como variable de entorno) que mapee cada categoría a su URL de feed RSS: 
Política:https://www.infobae.com/arc/outboundfeeds/rss/category/colombia/ Actualidad: https://qhubocali.com/feed/
Economía: https://www.valoraanalitik.com/feed/
Deportes: https://www.futbolred.com/rss 
Finanzas: https://feeds.elpais.com/mrss-s/list/ep/site/elpais.com/tag/finanzas_a
2. Implementar una función que itere sobre esta configuración, consumiendo cada feed RSS mediante una petición GET (ej. con axios). 
3. Para cada respuesta, usar una librería como xml2js para transformar el contenido XML a un objeto JSON, como el siguiente: 
{
  "newId": " el-banco-sabadell-cierra-las-cuentas-de-2025",
  "portalName": "Infobae",
  "newTitle": "Noticia de última hora",
  "newDate": "2026-02-09T10:30:00Z",
  "image": "https://www.example.com/images/noticia.jpg",
  "description": "Esta es una breve descripción de la noticia.",
  "content": "Contenido completo y detallado de la noticia.",
  "category": "Politica",
  "flag": false
} 
Para este caso Las propiedades adicionales a newId se deben extraer por cada item consultado al feed que corresponde a cada categoría, excepto category y flag, category será seteado de acuerdo el feed ejecutado y el flag será seteado por la base de datos con un valor false por defecto
4. Crear transformadores/mapeadores específicos para cada fuente (Infobae, Qhubocali, Valora, etc.) que estandaricen el objeto JSON resultante a un formato único que el sistema pueda procesar. Esto es crucial para gestionar las diferentes estructuras de los XML de cada portal.
Criterios de Aceptación
- La función puede consumir con éxito los 5 feeds RSS especificados. 
- El contenido XML de cada feed se transforma en un objeto JSON estructurado y predecible. 
- Se puede acceder a los items (noticias) dentro del JSON resultante.

Actualizar el nombre, slogan y logos del portal "La Crónica Nacional".
Descripción
Como desarrollador Frontend, quiero actualizar la identidad visual del portal de noticias, incluyendo el nombre, el slogan y los logos en el encabezado, para reflejar el nuevo branding "La Crónica Nacional" y asegurar que los elementos sean responsivos.
Responsable
Frontend
Tareas Sugeridas
1. Modificar el texto del nombre del portal a "La Crónica Nacional". 
2. Actualizar el texto del slogan a “Hechos que el poder no puede ocultar”. 
3. Reemplazar el archivo del logo principal actual por el nuevo logo propuesto. Adjunto nombre archivo Img-Logo.png . 
4. Añadir el nuevo "logo del slogan" en el encabezado (header) de la aplicación. Adjunto nombre archivo Img-Slogan.png. 
5. Aplicar los estilos CSS necesarios para que el "logo del slogan" sea responsive, asegurando su correcta visualización en diferentes dispositivos (móvil, tablet y escritorio).
6. Configurar el esquema de colores del portal de noticias con los encontrados en el logo y slogan, estos deben ser amigables al usuario.
Criterios de Aceptación
    • El encabezado del portal muestra el nombre "La Crónica Nacional" y el slogan “Hechos que el poder no puede ocultar”. 
    • El logo principal del portal ha sido reemplazado por el nuevo diseño. 
    • El logo del slogan se muestra correctamente en el encabezado. 
    • El layout del encabezado, incluyendo el nuevo logo del slogan, se adapta y visualiza correctamente en Chrome, Firefox y Safari, tanto en vistas de escritorio como en móvil, sin solapamientos ni elementos rotos.
    • EL color de todos los componentes debe ser compatible a nivel visual con los colores encontrados en el log y en el slogan.
